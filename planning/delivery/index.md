# IssueOps delivery records

This directory contains completed stage and planning-process delivery records.

A delivery record answers:

```text
What actually shipped?
What proved it?
What differed from the intended plan?
What boundaries were preserved?
What limitation or lesson should carry forward?
```

GitHub issues, pull requests, reviews, commits and workflow runs remain the canonical detailed audit trail. Delivery records link a curated authoritative subset rather than copying the full history.
