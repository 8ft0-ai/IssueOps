# Examples

These examples show how the execution-contract workflow should look in practice.

They are not a separate process. They are reference artefacts for the manual Stage 1 and Stage 1.1 workflow:

- [`execution-contract-example.md`](execution-contract-example.md) shows a good issue contract for a small agentic coding change.
- [`contract-verification-example.md`](contract-verification-example.md) shows how a reviewer can verify a pull request against the contract.

Use these examples when writing issues, planning Codex work or reviewing pull requests.
